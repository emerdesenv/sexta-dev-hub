# BASE DE COMANDOS PARA IA #
#############################

Objetivo:
Este arquivo serve como atalho de prompts para orientar a IA com contexto consistente, papéis claros e respostas acionáveis.

Regra de ouro:
Sempre que possível, use caminhos reais da pasta /ai e peça saída estruturada (plano, checklist, riscos, validação).

========================================
1) COMANDO MESTRE (recomendado para quase tudo)
========================================

Use a pasta /ai como base de contexto do projeto.

Siga esta ordem:
1. leia /ai/README_AI.md
2. leia /ai/context/AI_CONTEXT.md
3. classifique a tarefa com /ai/workflows/AI_TASK_ROUTER.md
4. orquestre os agentes com /ai/workflows/AI_AGENT_ORCHESTRATOR.md
5. siga a ordem de /ai/workflows/AGENT_EXECUTION_FLOW.md
6. respeite /ai/governance/AI_RULES.md, /ai/governance/CONSTRAINTS.md, /ai/context/ARCHITECTURE.md e /ai/context/PROJECT_MAP.md
7. entregue resposta consolidada, técnica e pronta para execução

Formato da resposta:
- entendimento da tarefa
- plano de execução
- riscos e mitigacoes
- validacao (testes/checklist)
- proximo passo recomendado

========================================
4) MELHOR USO NO DIA A DIA
========================================

1. Quando mudar de assunto tecnico, declare claramente:
"Mudando de contexto. Atue agora como /ai/agents/debugger.md"

2. Quando a conversa ficar longa, reset de contexto:
"Reinicie a analise usando /ai/agents/frontend-engineer.md e considere /ai como contexto do projeto."

3. Sempre inclua o tipo de saida esperado:
"Entregue em formato de checklist + plano de acao + testes de validacao."

4. Para evitar respostas vagas, adicione restricoes:
- stack/tecnologia
- arquivos impactados
- criterios de aceite
- prazo/prioridade

5. Para manter este arquivo atualizado:
- revisar mensalmente caminhos da pasta /ai
- remover comandos duplicados
- substituir referencias de arquivos renomeados
- incluir novos playbooks/workflows quando surgirem
