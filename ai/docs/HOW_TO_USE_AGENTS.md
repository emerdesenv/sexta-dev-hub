# How to Use Agents

Este documento explica como utilizar os agentes definidos no AI Engineering Framework.

Agentes representam papéis especializados dentro do processo de desenvolvimento.

---

# Conceito de agente

Um agente define:

- área de especialização
- responsabilidades
- limites
- arquivos de referência

Isso permite que a IA responda de forma mais focada.

---

# Principais agentes

## Architect

Responsável por decisões arquiteturais.

Usar quando:

- projetar novas funcionalidades
- definir integração entre serviços
- avaliar escalabilidade
- alterar estrutura do sistema

---

## Backend Engineer

Especialista em backend.

Usar quando:

- criar APIs
- implementar regras de negócio
- trabalhar com banco de dados
- integrar serviços

---

## Frontend Engineer

Especialista em interface.

Usar quando:

- criar telas
- desenvolver componentes
- aplicar design system
- trabalhar com UX

---

## Debugger

Especialista em investigação de erros.

Usar quando:

- algo não funciona
- ocorre erro em produção
- comportamento inesperado
- integração falhando

---

## Code Reviewer

Especialista em revisão de código.

Usar quando:

- revisar pull requests
- avaliar qualidade do código
- identificar melhorias

---

## Security Auditor

Especialista em segurança.

Usar quando:

- revisar autenticação
- avaliar upload de arquivos
- verificar validação de inputs
- identificar vulnerabilidades

---

## Test Engineer

Especialista em testes.

Usar quando:

- criar casos de teste
- planejar cobertura
- validar funcionalidades

---

## Refactor Specialist

Especialista em refatoração.

Usar quando:

- melhorar estrutura do código
- reduzir complexidade
- eliminar duplicação

---

## Product Designer

Especialista em UX e UI.

Usar quando:

- criar interfaces
- transformar design em componentes
- planejar experiência do usuário

---

# Exemplo de uso

Atue como `/ai/AGENTS/backend-engineer.md`.

Implemente um endpoint para upload de arquivos seguindo as regras do projeto.

---

# Boas práticas

- escolha o agente correto
- não misture responsabilidades
- use o orquestrador para tarefas complexas
